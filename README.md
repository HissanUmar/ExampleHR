# ExampleHR Time-Off Microservice

A NestJS microservice for managing employee time-off requests, local balance caching, and HCM synchronization for the ExampleHR platform.

This repository includes:
- A NestJS service with REST endpoints for balances, requests, sync, health, and metrics
- A defensive HCM adapter with retry-oriented behavior
- A local SQLite-backed persistence layer through TypeORM
- A mock HCM server used in tests to simulate balance changes and failure modes
- A comprehensive Jest test suite with coverage artifacts

## Architecture

- Pattern: reserve-then-confirm with defensive reconciliation
- Stack: NestJS, TypeORM, SQL.js, Jest, Supertest
- Core modules:
  - `BalancesModule` for balance reads, writes, and drift correction
  - `RequestsModule` for lifecycle management of time-off requests
  - `SyncModule` for batch sync and retry queue handling
  - `HcmModule` for the external HCM HTTP adapter
  - `AuthModule` for JWT and internal API key guards

## Features

- Cached leave balances by employee, location, and leave type
- Time-off request creation, approval, rejection, and cancellation
- Optimistic locking for local balance updates
- Batch balance ingestion from HCM
- Reconciliation and retry queue support
- JWT-protected application endpoints
- Internal API key protection for sync endpoints
- Mock HCM server for deterministic test scenarios

## Environment Variables

Create a `.env` file in the project root. A sample file is provided as `.env.example`.

| Variable | Description | Example |
| --- | --- | --- |
| `HCM_BASE_URL` | Base URL for the HCM API | `http://localhost:4100/hcm` |
| `HCM_API_KEY` | API key sent to HCM | `replace-me` |
| `HCM_TIMEOUT_MS` | HCM HTTP timeout in milliseconds | `5000` |
| `BALANCE_FRESHNESS_TTL_MIN` | Cache freshness threshold in minutes | `5` |
| `RECONCILE_CRON` | Reconciliation schedule | `0 */6 * * *` |
| `ALERT_DRIFT_DAYS` | Drift threshold for alerts | `1.0` |
| `CANCEL_GRACE_HOURS` | Grace period for approved cancellations | `24` |
| `JWT_SECRET` | JWT signing secret | `change-me` |
| `DB_PATH` | SQL.js database path | `./data/db.sqlite` |
| `INTERNAL_API_KEY` | Internal sync auth key | `internal-key` |

## API Endpoints

### Health and Metrics

- `GET /api/health` - service health and readiness
- `GET /api/metrics` - basic metrics output

### Balances

- `GET /api/balances/:employeeId` - list all balances for an employee
- `GET /api/balances/:employeeId/:locationId/:leaveType` - get one balance tuple

### Requests

- `POST /api/requests` - create a time-off request
- `GET /api/requests` - list requests with filters
- `GET /api/requests/:id` - get request details
- `PATCH /api/requests/:id/approve` - manager approval
- `PATCH /api/requests/:id/reject` - manager rejection
- `DELETE /api/requests/:id` - cancel a request

### Sync

- `POST /api/sync/trigger` - trigger reconciliation
- `GET /api/sync/failed-jobs` - process retry queue jobs
- `PUT /api/balances/batch` - ingest a full batch of HCM balances
